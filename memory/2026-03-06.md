## Important Lessons

### Keep PRs Separate by Feature
**Rule: Changes pertaining to different features should ALWAYS be separated into different PRs.**

This means:
- Trading calculator changes → one PR
- Memory section changes → another PR  
- Never mix unrelated features in the same branch
- Each PR should have a single, focused purpose

This makes:
- Review easier
- Rollback safer
- History cleaner
- Testing more focused
